# פקודה: `/new-project`

פקודה שמלווה את המשתמש בפתיחת פרויקט חדש מאפס, צעד אחרי צעד.

---

## התקנה

שמים את הקובץ `new-project.md` בתיקייה:

**Windows:**
```
C:\Users\<username>\.claude\commands\new-project.md
```

**Mac / Linux:**
```
~/.claude/commands/new-project.md
```

ברגע שהקובץ נמצא שם, הפקודה זמינה אוטומטית ב-Claude Code ללא צורך בהגדרות נוספות.

---

## איך מפעילים

בסביבת העבודה של Claude Code, הקלד:

```
/new-project
```

---

## מה קורה בכל שלב

**שלב 1: הבנת הפרויקט**
Claude שואל על מטרת הפרויקט, stack טכנולוגי, איפה ירוץ (local/cloud), ואם יש מגבלות עלות.

**שלב 2: יצירת CLAUDE.md**
קובץ הגדרות בתיקיית הפרויקט עם: סקירה כללית, stack, שלבי פיתוח, משתני סביבה, ואילוצי אבטחה.

**שלב 3: הגדרת GitHub**
הצעה ליצור repository לגיבוי ו-deployment, כולל הנחיות חיבור.

**שלב 4: מבנה תיקיות**
יצירת מבנה תיקיות מינימלי לפי הצרכים, ללא over-engineering.

**שלב 5: שמירה ל-MEMORY.md**
שמירת החלטות חשובות שהתגלו בשיחה לזיכרון הקבוע.

---

## מה הפקודה מייצרת

- קובץ `CLAUDE.md` בתיקיית הפרויקט
- מבנה תיקיות בסיסי
- GitHub repository (אם המשתמש בחר)

---

## כללים חשובים - אבטחה

**אסור** לבקש בצ'אט: tokens, API keys, סיסמאות, URLs של שירותים פרטיים (Railway, שרתים פרטיים, endpoints), או כל מידע רגיש אחר.
תמיד להנחות את המשתמש להכניס אותם ישירות לשירות הרלוונטי (קובץ `.env`, ממשק Railway, וכו').

- **למספר תמיד** את האפשרויות (1, 2, 3...) כדי שיהיה קל לבחור
- **להסביר בפשטות** - המשתמש לא בהכרח מפתח
